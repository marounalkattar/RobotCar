Car robot project

The car robot is a project based on an Arduino Uno controlling a car of two or four motors. It has many stages detailed down below.

Phase1: timing control robot

The purpose of this phase is to make sure that my robot component is in good condition and that the code is well written. In this phase the robot will go forward 2 second, backward 2 second, to the left 2 second, right 2 second and finally stop 2 second. In this stage, I have made 5 functions for each run (ex: void Forward() ) and this functions will make the work easier in the next stages. fonction code:

        void Forward() {
          digitalWrite(6, LOW);
          digitalWrite(7, HIGH);
          digitalWrite(8, LOW);
          digitalWrite(9, HIGH);
        }
        void Backward() {
          digitalWrite(6, HIGH);
          digitalWrite(7, LOW);
          digitalWrite(8, HIGH);
          digitalWrite(9, LOW);
        }
        
        void Left() {
          digitalWrite(6, LOW);
          digitalWrite(7, HIGH);
          digitalWrite(8, LOW);
          digitalWrite(9, LOW);
        }
        
        void Right() {
          digitalWrite(6, LOW);
          digitalWrite(7, LOW);
          digitalWrite(8, LOW);
          digitalWrite(9, HIGH);
        }
        
        void Stop() {
          digitalWrite(6, LOW);
          digitalWrite(7, LOW);
          digitalWrite(8, LOW);
          digitalWrite(9, LOW);
        
        }
        
                  

Phase2: PC communication

The robot car will take order from the Serial monitor so when I press w or W the car will go forward, X or x the car will stop, when pressing s or S the car will go backward, A or a the car will move to the left and D or d the car will go to the right.

Phase3: Smartphone control

I have made an android application to control the robot. Watch the video of the application interface. The smartphone and the robot are connected via Bluetooth using hc-05 sensor.

        

Phase4: Accelerometer control

The robot will be controlled by the accelerometer sensor of my smartphone. This sensor is an X, Y and Z sensor system and will measure the orientation of my phone by giving orders to the Arduino to move the car.

Phase5: Voice control

I created an application to manipulate my robot over voice control system so when I say “go forward” the robot must go forward.

Phase6: Black and White line follower

In this stage the robot will follow a colored line. First, the line will be black and then it will be white.

        

Phase7: Object Avoid

An ultra-sonic hc-sr04 sensor will control the robot, so the car won’t smash up with obstacles. So here I have updated the car with putting the hc-sr04 sensor on servo and then when the distance is less than 30 cm the servo will rotate the sensor to the right and then to left to take distance. Then the controller will calculate where the distance is bigger left and right to rotate the car.

Phase8: Edge avoiding

Finally, I will put the robot on the table so I can test its ability to avoid edges.

Projet de robot de voiture

Le robot voiture est un projet basé sur Arduino Uno contrôlant une voiture de deux ou quatre moteurs. Ce projet comporte de nombreuses étapes détaillées ci-dessous.

Phase 1 : robot de contrôle de chronométrage

Le but de cette phase est de s’assurer que les composant du robot est en bon état et que le code est bien écrit. Dans cette phase, le robot avancera 2 secondes, reculera 2 secondes, gauche 2 secondes, droite 2 secondes et enfin s’arrêtera 2 secondes. Dans cette étape, j’ai créé 5 fonctions pour chaque exécution (ex: void Forward()) et ces fonctions faciliteront le travail dans les étapes suivantes. Code des fonctions : 

          void Forward() {
            digitalWrite(6, LOW);
            digitalWrite(7, HIGH);
            digitalWrite(8, LOW);
            digitalWrite(9, HIGH);
          }
          void Backward() {
            digitalWrite(6, HIGH);
            digitalWrite(7, LOW);
            digitalWrite(8, HIGH);
            digitalWrite(9, LOW);
          }
          
          void Left() {
            digitalWrite(6, LOW);
            digitalWrite(7, HIGH);
            digitalWrite(8, LOW);
            digitalWrite(9, LOW);
          }
          
          void Right() {
            digitalWrite(6, LOW);
            digitalWrite(7, LOW);
            digitalWrite(8, LOW);
            digitalWrite(9, HIGH);
          }
          
          void Stop() {
            digitalWrite(6, LOW);
            digitalWrite(7, LOW);
            digitalWrite(8, LOW);
            digitalWrite(9, LOW);
          
          }
          
            

Phase 2: Communication par PC

La voiture robot prendra l’ordre du moniteur série, donc lorsque j’appuie sur w ou W, la voiture avance, X ou x la voiture s’arrête, en appuyant sur s ou S, la voiture recule, A ou a, la voiture se déplace vers la gauche et D ou d, la voiture va à droite.

Phase 3 : Contrôle à l'aide smartphone

J’ai fait une application android pour contrôler le robot. Regardez la vidéo de l’interface de l’application. Le smartphone et le robot sont connectés via Bluetooth à l’aide d'un module Bluetooth "hc-05".

        

Phase 4 : Contrôle de l’accéléromètre

Le robot sera contrôlé par le capteur accéléromètre de mon smartphone. Ce capteur est un système de capteurs X, Y et Z et mesurera l’orientation de mon téléphone en donnant des ordres à l’Arduino de déplacer la voiture.

Phase 5 : Commande vocale

J’ai créé une application pour manipuler mon robot sur le système de commande vocale donc quand je dis « aller de l’avant » le robot doit aller de l’avant.

Phase 6 : Suiveur de ligne noir et blanc

Dans cette étape, le robot suivra une ligne colorée. D'abord, la ligne sera noire et ensuite elle sera blanche.

        

Phase 7 : Éviter les objets

Un capteur hc-sr04 à ultrasons contrôlera le robot, afin que la voiture ne se heurte pas à des obstacles. Donc, ici, j’ai mis à jour la voiture en mettant le capteur hc-sr04 sur servo, puis lorsque la distance est inférieure à 30 cm, le servo tournera le capteur vers la droite puis vers la gauche pour prendre de la distance. Ensuite, le contrôleur calculera où la distance est plus grande à gauche et à droite pour faire pivoter la voiture.

Phase 8 : Éviter les bords

Enfin, je vais poser le robot sur la table afin de pouvoir tester sa capacité à éviter les bords.
